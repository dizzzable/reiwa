Конструктор бота, PW1 — кастомные эмодзи в подписях кнопок. Bot-only (rezeis не трогали).

### ✨ Кастомные/премиум-эмодзи на кнопках
- Подписи inline-кнопок теперь прогоняются через `renderButtonLabel`: токены `{{KEY}}` и `:slug:` из паков больше не утекают в кнопку буквально (раньше было видно `:widestdicons_2_by_fstikbot_18:`).
- Если подпись начинается с известного `:slug:`/`{{KEY}}` с premium-id и у владельца бота есть Premium — токен поднимается в `icon_custom_emoji_id` кнопки (Bot API 9.4), а текст очищается. Это единственный способ показать premium-эмодзи на кнопке (в самом тексте кнопки Telegram не поддерживает custom_emoji-entity).
- Иначе токен заменяется на fallback-глиф пака (или unicode для `{{KEY}}`). Неизвестные `:slug:` остаются как есть (литерал).
- Подключено на всех поверхностях с операторскими кнопками: главное меню (`main-keyboard`), динамические экраны (`screen-renderer`), reply-/welcome-экран, hub «Пригласить» (referral + partner), help/rules.
